# Security Policy

## Reporting a Vulnerability

**Please do not file a public GitHub issue for security vulnerabilities.**

Public issues are visible to everyone immediately. Filing a security bug publicly could expose users to risk before a fix is available.

Instead, please report security vulnerabilities by emailing:

**support@pagesets.co**

Use the subject line: `Security Vulnerability Report`

---

## What to Include

A good security report helps us reproduce and fix the issue quickly. Please include:

- **Description** — what the vulnerability is and what it affects
- **Steps to reproduce** — as specific as possible
- **Impact** — what could an attacker do with this?
- **Environment** — Chrome version, OS, PageSets version
- **Proof of concept** — if you have one (optional but helpful)

---

## What to Expect

| Timeframe | What happens |
|---|---|
| **Within 48 hours** | We acknowledge receipt of your report |
| **Within 7 days** | We assess severity and confirm whether it's a valid vulnerability |
| **Within 30 days** | We aim to have a fix released for confirmed issues |

We'll keep you informed throughout the process and credit you in the release notes if you'd like.

---

## Scope

PageSets is a Chrome extension. The following are in scope for security reports:

- The PageSets Chrome extension itself
- The pagesets.co website
- The Pro license validation flow (LemonSqueezy integration)

The following are **out of scope:**

- Vulnerabilities in third-party services (LemonSqueezy, Chrome itself, etc.)
- Issues that require physical access to a user's device
- Social engineering attacks
